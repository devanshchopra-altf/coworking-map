1. Initial Setup & Deployment

Follow these steps once to get the app live for your team.

1

Get a Google Maps API Key

  1. Go to console.cloud.google.com and sign in with a Google account
  2. Click Select a project → New Project → give it a name → Create
  3. In the left menu go to APIs & Services → Library
  4. Search for and enable Maps JavaScript API
  5. Search for and enable Geocoding API (required for adding buildings)
  6. Go to APIs & Services → Credentials → Create Credentials → API Key
  7. Copy the key — it looks like AIzaSy…
⚠️ Both APIs must be enabled. Maps JavaScript API renders the map; Geocoding API converts building names to coordinates when you add new buildings.
2

Set Up the Project Locally

  1. Unzip the downloaded coworking-map.zip
  2. Open a terminal in the folder
  3. Copy the env file and add your key:
cp .env.example .env
# Open .env and replace "your_google_maps_api_key_here" with your key

Your .env file should look like:

VITE_GOOGLE_MAPS_API_KEY=AIzaSyABC123xyz...
  • Install and run:
    • npm install
npm run dev

    Open http://localhost:5173 — you should see the map with all Delhi NCR buildings.

    3

    Deploy to Vercel (Share with Team)

    1. Create a free account at github.com if you don't have one
    2. Create a new repository and push the project folder to it: 
      git init
git add .
git commit -m "Initial commit"
git remote add origin https://github.com/YOUR_USERNAME/coworking-map.git
git push -u origin main
    3. Go to vercel.com → Sign up / Log in → Add New Project
    4. Import your GitHub repository
    5. In the Environment Variables section, add:
      • Key: VITE_GOOGLE_MAPS_API_KEY
      • Value: your API key
    6. Click Deploy
    7. Share the live URL (e.g. https://coworking-map.vercel.app) with your team
    4

    Restrict Your API Key (Recommended)

    To prevent unauthorised use of your API key:

    1. Go to Google Cloud Console → APIs & Services → Credentials
    2. Click your API key → Edit
    3. Under Application restrictions, select HTTP referrers
    4. Add your Vercel URL: https://your-app-name.vercel.app/*
    5. Also add http://localhost:5173/* for local development
    6. Click Save

    2. Navigating the Map

    City Tab Bar (top of screen)

    Click any city tab to instantly jump the map to that city. The sidebar will update to show only buildings in that city, and the submarket filter will reset to show all submarkets for that city.

    • All Cities — zooms out to show all of India with every building
    • Delhi NCR — your pre-loaded dataset of 507 buildings
    • Mumbai / Bangalore / etc. — empty until you add buildings there

    Map Controls

    • Scroll / pinch — zoom in and out
    • Click + drag — pan the map
    • Click a marker — opens a popup with full building details
    • Click anywhere on the map — closes any open popup
    • ◀ / ▶ button (top left of map) — hides/shows the sidebar for more map space

    Understanding Markers

    • Coloured dot markers — pre-loaded buildings from your Excel dataset, colour = submarket
    • Markers with a + symbol — buildings you've manually added
    • The legend strip at the top of the map shows which colour = which submarket

    3. Searching for Buildings

    Use the search bar at the top of the left sidebar.

    1. Type any part of a building name, micromarket, or submarket
    2. The sidebar list and map markers update instantly as you type
    3. Click any result in the sidebar — the map flies to that building and opens its details popup
    4. Press ↺ Reset to clear the search and restore all markers
    💡 Tip: Search is not case-sensitive. Searching "dlf" will match "DLF Cyber City", "DLF Tower", etc.

    4. Using Submarket Filters

    The submarket filter chips are in the sidebar below the search bar.

    • Click All to show all submarkets
    • Click any submarket chip to toggle it on/off — its markers appear/disappear on the map
    • You can have multiple submarkets active simultaneously
    • The number next to each chip shows how many buildings are in that submarket
    • Submarket colours match the map marker colours
    💡 Tip: To focus on one submarket, click All first to deselect all, then click just the submarket you want. (Currently clicking All re-selects all — deselect by clicking individual chips.)

    5. Adding a Single Building

    Use this when you need to add one building at a time with full details.

    1

    Open the Add Buildings modal

    Click the ➕ Add Buildings button in the top-right corner of the page.

    2

    Select "Single Building" tab

    The modal opens on Single Building mode by default.

    3

    Fill in the form

    Required fields are marked with *:

    • Building Name * — be as specific as possible. E.g. "Bagmane Tech Park" not just "Bagmane"
    • Micromarket * — the locality/area. E.g. "CV Raman Nagar" or "Whitefield"
    • City * — select from the dropdown
    • Grade, Type, Category, Year, Area, Developer, Rent — optional but recommended for richer popups
    💡 The more specific the building name and micromarket, the more accurately it will be placed on the map. The geocoder tries: Building Name + Micromarket + City first, then Micromarket + City as fallback.
    4

    Click "Add Building"

    The app will geocode the building (calls Google Maps to find its coordinates). A status message shows:

    • ✅ high accuracy — building found precisely
    • medium accuracy — placed at micromarket level
    • low accuracy — only city-level placement

    Once added, the building appears on the map immediately with a + marker and an ADDED badge in the sidebar.

    ⚠️ Added buildings are saved in your browser's local storage. They persist across page refreshes but are tied to your browser/device. To share with the whole team, export and update the base data file (see Section 9).

    6. Bulk Adding Buildings (CSV)

    Use this to add 5, 20, or 100+ buildings at once.

    1

    Open Add Buildings → Bulk Add tab

    Click ➕ Add Buildings → click 📋 Bulk Add (CSV) tab.

    2

    Prepare your list

    Each line follows the format: Building Name, Micromarket, City

    RMZ Millenia Business Park, OMR, Chennai Bagmane Tech Park, CV Raman Nagar, Bangalore One BKC, Bandra Kurla Complex, Mumbai Raheja Mindspace, HITEC City, Hyderabad Magarpatta Cybercity, Hadapsar, Pune Embassy Manyata Business Park, Hebbal, Bangalore
    💡 You can prepare this list in Excel, copy the relevant columns, and paste directly. Lines starting with # are treated as comments and ignored.
    3

    Paste and submit

    Paste your list into the text area. The app shows how many valid rows it detected. Click Add X Buildings.

    Each building is geocoded one by one. You'll see a live progress list — green = success, red = failed. Failed buildings can be added manually afterwards with the single building form.

    4

    Review the results

    Successfully geocoded buildings appear on the map immediately. Close the modal and navigate to the relevant city to verify placement.

    ⚠️ Geocoding uses your Google Maps API key and counts towards your usage quota. Google provides $200/month free which covers ~40,000 geocoding requests — more than enough for typical use.

    7. Managing Cities

    Pre-loaded Cities

    These cities are available by default in the tab bar: Delhi NCR, Mumbai, Bangalore, Hyderabad, Pune, Chennai.

    Adding a New City

    1. Click ➕ Add Buildings
    2. At the bottom of the modal, click + Add new city
    3. Type the city name (e.g. "Kolkata") and press Enter or click Add
    4. The new city immediately appears as a tab in the top bar
    5. You can now add buildings to that city using the Single or Bulk add forms
    💡 New cities are saved to localStorage and persist across sessions for that browser.

    8. Removing Buildings

    You can only remove buildings that you manually added (marked with ADDED). The original pre-loaded dataset cannot be deleted from the UI.

    1. Click the + marker on the map or click the building in the sidebar
    2. In the popup, click 🗑 Remove this building
    3. Confirm the deletion

    9. Updating the Base Dataset

    The base 507-building Delhi NCR dataset lives in src/data/buildings.json. To permanently add/edit buildings for the whole team:

    1

    Option A — Edit the JSON directly

    Open src/data/buildings.json and add entries. Each building object must have at minimum:

    {
  "name": "Building Name",
  "location": "Micromarket Area",
  "submarket": "Submarket Name",
  "city": "Mumbai",
  "lat": 19.0596,
  "lng": 72.8295,
  "isCustom": false
}

    Then push to GitHub — Vercel auto-redeploys in ~30 seconds.

    2

    Option B — Export from the App + Update JSON

    After adding buildings via the UI, you can retrieve them from your browser's local storage and paste into the JSON:

    1. Open browser DevTools (F12) → Application → Local Storage
    2. Find the key cwmap_custom_buildings — copy its value
    3. Paste and merge into src/data/buildings.json
    4. Set "isCustom": false for all entries
    5. Push to GitHub

    10. FAQ & Troubleshooting

    ❓ The map shows a grey screen / "Map failed to load"

    Your API key is missing or incorrect. Check:

    • In local dev: does .env exist and have the correct key?
    • On Vercel: is VITE_GOOGLE_MAPS_API_KEY set in Environment Variables?
    • Is Maps JavaScript API enabled in Google Cloud Console?
    • If you restricted the key by HTTP referrer, is the current domain listed?

    ❓ A building geocoded but landed in the wrong place

    The geocoder placed it at micromarket or city level instead of the exact building. To fix:

    1. Remove the building (click the + marker → Remove)
    2. Re-add it with a more specific name. E.g. instead of "Tower 1" use "DLF Cyber City Tower 1 Gurgaon"

    ❓ Geocoding fails / "Could not geocode"

    • Check that Geocoding API is enabled in Google Cloud Console
    • Check you haven't hit Google's daily geocoding quota (unlikely unless doing thousands of buildings)
    • Try a different building name spelling

    ❓ My added buildings disappeared after refreshing

    Added buildings are stored in your browser's localStorage. They disappear if you:

    • Clear your browser cache/cookies
    • Use a different browser or device
    • Use Incognito/Private mode

    To make buildings permanent for the whole team, follow the steps in Section 9 to add them to the JSON file.

    ❓ How do I add a completely new city not in the list?

    Open Add Buildings modal → click + Add new city at the bottom → type the city name → press Enter. It will appear as a tab immediately.

    ❓ Can multiple team members add buildings simultaneously?

    Not in real-time — each person's added buildings are stored locally in their own browser. To synchronise additions across the team, the workflow is: add buildings in UI → export from localStorage → merge into buildings.json → push to GitHub → Vercel redeploys → everyone gets the update on next refresh.

    For real-time collaboration in the future, the app can be connected to Google Sheets as a backend — reach out if you need this upgrade.

    📬 Need help? If you encounter any issues or want to add new features (real-time Google Sheets sync, coworking operator overlay, heat maps, export to PDF), the app is designed to be easily extended.
